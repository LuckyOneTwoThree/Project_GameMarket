## 全局视角：资产的生命周期

在 Game Market 中，(在当前的定义下)一件饰品会经历三种状态流转：

实物 ——> 凭证 （充值入库）

凭证 ——> 资金 （撮合交易）

凭证 ——> 实物 （提现取回）

## 二、核心业务场景详解

目前重点关注于"Steam-CS饰品"交易的相关场景

### 模块 1：C 端交易闭环

#### **场景 1.1：充值入库**

- **用户故事**：Lucky想把 Steam 里的“AK-47 | 红线”饰品存入平台，以便出售。
- **业务流**：
  - **选货**：用户在【资产中心】（或者【交易大厅】的快捷入口）点击“存入”，系统拉取其 Steam 库存。
  - **安检**：系统调用 match_rules 自动校验 SKU（磨损是否在 T1/T2 区间？贴纸是否太贵？）。
  - **移交**：校验通过 ——> Bot 发起报价 ——> Lucky在 Steam 确认
  - **上链**：Bot 确认收货 ——> Lucky账户中增加“AK-47 | 红线”凭证。

#### **场景 1.2：极速买入**

- **用户故事**：用户Lucky想买一个 AK-47 皮肤，希望买了就能立刻在平台内再次交易。
- **业务流**：
  - **筛选**：在交易大厅勾选 `[x] 仅匹配极速交付`。
  - **下单**：点击“市价买入”。
  - **交割**：
    1. 撮合引擎匹配到 Bot 托管的卖单（通常由 MM 挂出）。
    2. 资金划转，**资产凭证**瞬间转移给小李。
    3. **状态****继承**：如果该饰品还在 Steam 7天冷却期内，小李获得凭证但不可提现；如果冷却已结束，小李可选择提现。

#### **场景 1.3：资产取回**

- **用户故事**：Lucky买到了心仪的皮肤饰品，想提取到自己的 Steam 账号里在游戏中使用。

**业务流**：

1. **申请**：在【资产中心】选中状态为“可用 ”的饰品，点击“取回”。
2. **锁定**：系统冻结账户内的资产凭证，并锁定 Bot 数据库中的对应 Asset ID。
3. **交付**：Bot 发起“单向赠送报价”给小李的 Steam Trade URL。
4. **销账**：小李在 Steam 接受报价 ——> 平台销毁凭证 ——> 扣除物理库存。

### 模块 2：B 端流动性供给

#### **场景 2.1：做市商铺货**

- **用户故事**：量化团队通过程序化脚本为市场提供深度。
- **业务流**：
  - **备货**：通过 API 批量将 1000 个“变革武器箱”充值进 Bot 集群。
  - **挂单**：读取价格精度，挂出阶梯卖单。
  - **调价**：监测到外部市场（Buff/Steam）价格变动，取消旧单并请求新单。
  - **库存风控**：若遇到单边买入行情（库存快卖光了），触发警报，自动补货或停止挂单。

#### **场景 2.2：****P2P** **挂单出售**

- **用户故事**：Lucky有一个价值 2000 USDT 的“潘多拉手套”饰品，属于高危资产，选择自己保存并挂单。
- **业务流**：
  - **上架**：在【挂单管理】选择手套，设定价格。系统识别路由，无需充值进 Bot。
  - **响应**：买家下单  ——> 提醒卖家（方式很多）。
  - **交付**：Lucky 点击“一键交付” ——> 跳转 Steam 确认发送报价。
  - **结算**：系统确认后资金转入Lucky账户。

#### **场景 2.3：****P2P** **异常处理**

- **用户故事**：Lucky尝试发送饰品，但发现买家 Steam 账户被限制交易（红锁），导致无法送达。
- **业务流**：
  - **报错**：Steam API 返回错误码。
  - **申诉**：老张在工作台点击“异常申诉”，上传 Steam 报错截图。
  - **裁决**：后台运营审核通过   ——> 订单取消 ——> 判定买家全责  ——> 赔付违约金

### 模块 3：后台风控与运维

#### **场景 3.1：全站熔断**

- **用户故事**：Steam 服务器周二例行维护，或者 API 出现大面积波动，Bot 交付连续失败。
- **业务流**：
  - **监控**：系统检测到 Bot 交付成功率跌破 20%。
  - **熔断**：自动触发熔断机制。
  - **降级**：前端隐藏相关图标，所有新订单强制走 P2P 匹配，或者暂停受影响 SKU 的交易。
  - **恢复**：API 恢复正常 10 分钟后，自动解除熔断。

#### **场景 3.2：防双花人工审计**

- **用户故事**：Bot 发送饰品超时，系统尝试取消订单失败（状态未知），资金处于冻结状态。
- **业务流**：
  - **挂起**：系统将该订单进行标记，推送到【纠纷仲裁台】。
  - **核查**：
    1. 运营**无需登录** Bot 账号。
    2. 点击“查询快照”，系统后端调用 Steam API 获取该 Bot 的历史记录并展示在后台。
  - **操作**：
    1. 若饰品还在 Bot 库中：点击后台“强制退款”。
    2. 若饰品已转移给买家：点击后台“强制成交（补单）”。

#### 场景 3.3：商户穿仓清算

- **用户故事**：P2P 商户“Lucky”连续违约，保证金已被罚扣至低于安全线。
- **业务流**：
  - **预警**：保证金 < 50 USDT，系统自动发送短信/邮件催缴。
  - **强平**：保证金 < 10 USDT（不足以支付一次违约金）。
  - **执行**：
    1. 系统触发紧急终止开关。
    2. 强制下架该商户所有挂单。
    3. 冻结其资金账户的提现功能。
  - **通知**：向商户发送“服务终止通知书”。
