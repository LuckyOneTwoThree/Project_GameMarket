# 测试策略

## 测试框架

### 前端测试

1. **单元测试**

- **框架**：Jest + React Testing Library
- **测试范围**：组件功能、Hook 逻辑、工具函数
- **示例**：

```javascript
// 组件测试示例
import { render, screen, fireEvent } from '@testing-library/react';
import TradePanel from './TradePanel';

describe('TradePanel', () => {
  test('renders trade panel with correct symbol', () => {
    render(<TradePanel symbol="AKRED-FT-T1" />);
    expect(screen.getByText('AK-47 | Redline')).toBeInTheDocument();
  });

  test('handles buy order submission', () => {
    const onSubmit = jest.fn();
    render(<TradePanel symbol="AKRED-FT-T1" onSubmit={onSubmit} />);
    
    fireEvent.change(screen.getByLabelText('Price'), { target: { value: '100' } });
    fireEvent.change(screen.getByLabelText('Quantity'), { target: { value: '1' } });
    fireEvent.click(screen.getByText('Buy'));
    
    expect(onSubmit).toHaveBeenCalledWith({ side: 'buy', price: 100, quantity: 1 });
  });
});
```

2. **集成测试**

- **框架**：Cypress
- **测试范围**：页面导航、表单提交、API 调用
- **示例**：

```javascript
// 集成测试示例
```

3. **端到端测试**

- **框架**：Cypress
- **测试范围**：完整用户流程，从登录到交易完成
- **示例**：

```javascript
// 端到端测试示例
describe('Complete trade flow', () => {
  it('allows user to deposit asset and trade', () => {

```

### 后端测试

1. **单元测试**

- **框架**：Jest
- **测试范围**：服务层逻辑、工具函数、数据模型
- **示例**：

```javascript
// 服务层测试示例
import tradeService from '../services/trade';
import orderModel from '../models/order';

jest.mock('../models/order');

describe('tradeService', () => {
  test('creates order successfully', async () => {
    const orderData = {
      userId: 1,
      skuId: 1,
      side: 'buy',
      type: 'limit',
      price: 100,
      quantity: 5
    };
    
    const mockOrder = { id: 1, ...orderData, status: 'pending' };
    orderModel.create.mockResolvedValue(mockOrder);
    
    const result = await tradeService.createOrder(orderData);
    expect(result).toEqual(mockOrder);
    expect(orderModel.create).toHaveBeenCalledWith(orderData);
  });
});
```

2. **集成测试**

- **框架**：Jest + Supertest
- **测试范围**：API 接口、数据库操作、业务流程
- **示例**：

```javascript
// API 测试示例
import request from 'supertest';
import app from '../app';
import orderModel from '../models/order';

jest.mock('../models/order');

describe('/api/orders', () => {
  test('POST /api/orders creates order', async () => {
    const orderData = {
      symbol: 'AKRED-FT-T1',
      side: 'buy',
      type: 'limit',
      price: 100,
      quantity: 5
    };
    
    const mockOrder = { id: 1, ...orderData, status: 'pending' };
    orderModel.create.mockResolvedValue(mockOrder);
    
    const response = await request(app)
      .post('/api/orders')
      .set('Authorization', 'Bearer test-token')
      .send(orderData);
    
    expect(response.status).toBe(201);
    expect(response.body).toEqual(mockOrder);
  });
});
```

3. **数据库测试**

- **框架**：Jest + Sequelize Test Helpers
- **测试范围**：数据库模型、关系、约束
- **示例**：

```javascript
// 数据库测试示例
import { sequelize, User, Wallet } from '../models';

beforeAll(async () => {
  await sequelize.sync({ force: true });
});

describe('User model', () => {
  test('creates user with wallet', async () => {
    const user = await User.create({
      username: 'testuser',
      email: 'test@example.com',
      passwordHash: 'hashed-password'
    });
    
    const wallet = await Wallet.create({
      userId: user.id,
      currency: 'USDT',
      balance: 1000
    });
    
    const userWithWallet = await User.findByPk(user.id, {
      include: Wallet
    });
    
    expect(userWithWallet.wallets.length).toBe(1);
    expect(userWithWallet.wallets[0].currency).toBe('USDT');
  });
});
```

## 性能测试

### 前端性能测试

1. **加载性能**

- **工具**：Lighthouse
- **指标**：
  - First Contentful Paint (FCP) < 1.5s
  - Largest Contentful Paint (LCP) < 2.5s
  - Cumulative Layout Shift (CLS) < 0.1
  - Time to Interactive (TTI) < 3.8s

2. **交互性能**

- **工具**：Chrome DevTools Performance
- **指标**：
  - 首次输入延迟 (FID) < 100ms
  - 累计首次输入延迟 (CUMULATIVE LAYOUT SHIFT) < 0.1
  - 交互到下一次绘制 (INP) < 200ms

3. **资源使用**

- **工具**：Chrome DevTools Memory
- **指标**：
  - 内存使用 < 500MB
  - 无内存泄漏

### 后端性能测试

1. **API 性能**

- **工具**：Artillery
- **测试场景**：
  - 单接口并发测试：1000 并发用户，持续 30 秒
  - 混合场景测试：模拟真实用户操作流程
- **指标**：
  - 平均响应时间 < 200ms
  - 95th 百分位响应时间 < 500ms
  - 错误率 < 1%

2. **数据库性能**

- **工具**：pgbench
- **测试场景**：
  - 并发查询测试
  - 批量插入测试
  - 复杂查询测试
- **指标**：
  - 查询响应时间 < 100ms
  - 事务处理能力 > 1000 TPS

3. **系统性能**

- **工具**：JMeter
- **测试场景**：
  - 系统负载测试：逐步增加并发用户数
  - 系统稳定性测试：持续 24 小时运行
- **指标**：
  - CPU 使用率 < 70%
  - 内存使用率 < 80%
  - 系统稳定运行 24 小时无故障

## 安全测试

### 前端安全测试

1. **XSS 测试**

- **工具**：OWASP ZAP
- **测试方法**：
  - 注入恶意脚本到输入字段
  - 测试 URL 参数注入
  - 测试 Cookie 操作

2. **CSRF 测试**

- **工具**：OWASP ZAP
- **测试方法**：
  - 模拟跨站请求伪造攻击
  - 测试 CSRF Token 有效性

3. **敏感信息测试**

- **工具**：Chrome DevTools
- **测试方法**：
  - 检查 localStorage 和 sessionStorage 中的敏感信息
  - 检查网络请求中的敏感信息

### 后端安全测试

1. **SQL 注入测试**

- **工具**：OWASP ZAP
- **测试方法**：
  - 注入 SQL 代码到输入字段
  - 测试参数化查询有效性

2. **认证测试**

- **工具**：OWASP ZAP
- **测试方法**：
  - 测试 JWT 令牌安全性
  - 测试密码重置流程
  - 测试权限绕过

3. **API 安全测试**

- **工具**：OWASP ZAP
- **测试方法**：
  - 测试 API 访问控制
  - 测试速率限制
  - 测试数据验证

## 测试环境

### 开发环境测试

- **配置**：本地开发环境
- **目的**：快速反馈，开发过程中进行单元测试
- **工具**：Jest、React Testing Library

### 测试环境测试

- **配置**：独立的测试环境，与生产环境相似
- **目的**：集成测试，验证系统各组件协同工作
- **工具**：Cypress、Supertest、Artillery

### 预生产环境测试

- **配置**：与生产环境相同配置
- **目的**：最终验证，确保系统在生产环境配置下正常运行
- **工具**：Lighthouse、JMeter、OWASP ZAP

## 测试流程

### 开发阶段

1. **单元测试**：开发人员在编写代码时编写单元测试
2. **代码审查**：提交代码前进行代码审查，确保测试覆盖率
3. **CI 测试**：代码提交后，CI 系统自动运行单元测试和集成测试

### 发布阶段

1. **预发布测试**：在预生产环境进行全面测试
2. **性能测试**：验证系统性能是否满足要求
3. **安全测试**：进行安全扫描，确保系统安全
4. **发布决策**：根据测试结果决定是否发布

### 维护阶段

1. **回归测试**：每次代码变更后运行回归测试
2. **监控测试**：定期运行监控测试，确保系统正常运行
3. **安全更新测试**：安全补丁发布后进行测试

## 测试自动化

### CI/CD 集成

1. **GitHub Actions**

```yaml
# .github/workflows/test.yml
name: Test

on:
  push:
    branches:
      - develop
      - main
  pull_request:
    branches:
      - develop
      - main

jobs:
  frontend-tests:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Set up Node.js
        uses: actions/setup-node@v3
        with:
          node-version: '18'
      - name: Install dependencies
        run: |
          cd frontend
          npm install
      - name: Run tests
        run: |
          cd frontend
          npm test
      - name: Check coverage
        run: |
          cd frontend
          npm run coverage

  backend-tests:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Set up Node.js
        uses: actions/setup-node@v3
        with:
          node-version: '18'
      - name: Install dependencies
        run: |
          cd backend
          npm install
      - name: Run tests
        run: |
          cd backend
          npm test
      - name: Check coverage
        run: |
          cd backend
          npm run coverage
```

## 测试策略总结

通过建立全面的测试策略，包括单元测试、集成测试、端到端测试、性能测试和安全测试，可以确保 Game Market 平台的质量和可靠性。测试覆盖率目标和性能指标的设定，为开发团队提供了明确的质量标准。测试自动化和 CI/CD 集成，确保了测试的及时性和有效性。

测试策略的实施将有助于：

1. **提高代码质量**：通过单元测试和集成测试，确保代码逻辑正确
2. **降低系统风险**：通过性能测试和安全测试，确保系统稳定和安全
3. **加速开发周期**：通过自动化测试，快速反馈开发问题
4. **提升用户体验**：通过前端性能测试，确保页面加载和交互流畅
5. **保障交易安全**：通过安全测试，确保用户资产安全

测试策略是 Game Market 平台成功的重要保障，将贯穿整个开发和运维过程。
