# 技术栈与架构

## 前端技术栈

### 框架与库
- **主框架**：React 18+，使用函数组件和 Hooks
- **状态管理**：Redux Toolkit，管理全局状态
- **UI 组件库**：Ant Design Pro，提供丰富的组件和布局
- **图表库**：ECharts，用于交易深度图和价格走势图
- **网络请求**：Axios，处理 API 调用
- **WebSocket**：Socket.io-client，处理实时数据推送
- **路由**：React Router v6，管理页面路由
- **国际化**：i18next，支持多语言

### 构建工具
- **打包工具**：Vite，提供快速的开发体验和优化的生产构建
- **代码规范**：ESLint + Prettier，确保代码质量
- **类型检查**：TypeScript，提供类型安全

## 后端技术栈

### 语言与框架
- **主语言**：Node.js 18+
- **Web 框架**：Express.js，构建 RESTful API
- **WebSocket 服务**：Socket.io，提供实时数据推送
- **认证**：JWT，实现无状态认证
- **加密**：bcrypt，处理密码加密

### 数据库
- **主数据库**：PostgreSQL，存储用户数据、订单数据、资产数据等
- **缓存**：Redis，缓存热点数据、管理 Session/Token、实现分布式锁
- **消息队列**：RabbitMQ，处理异步任务（如订单结算、资产交割）

### 工具库
- **ORM**：Sequelize，简化数据库操作
- **日志**：Winston，记录系统日志
- **验证**：Joi，验证请求参数
- **定时任务**：node-cron，执行定时任务（如价格更新、清算）

## 基础设施

### 服务器
- **云服务**：AWS 或阿里云
- **容器化**：Docker + Kubernetes，实现服务编排和自动扩缩容

### 监控与告警
- **监控**：Prometheus + Grafana，监控系统性能和业务指标
- **告警**：Alertmanager，设置告警规则并通知
- **日志管理**：ELK Stack（Elasticsearch + Logstash + Kibana），集中管理和分析日志

## 架构设计

### 微服务架构
- **API 网关**：统一入口，处理认证、路由、限流等
- **用户服务**：处理用户注册、登录、信息管理
- **交易服务**：处理订单创建、撮合、结算
- **资产服务**：处理资产充值、提现、库存管理
- **风控服务**：处理风险控制、熔断机制
- **通知服务**：处理系统通知、消息推送

### 数据流
1. **前端**：用户操作 → API 调用 → WebSocket 订阅
2. **后端**：API 请求 → 业务逻辑处理 → 数据库操作 → WebSocket 推送
3. **外部集成**：Steam API 调用 → 数据处理 → 业务逻辑更新

### 安全设计
- **HTTPS**：全站点 HTTPS 加密
- **CORS**：配置跨域资源共享
- **Rate Limiting**：API 访问速率限制
- **SQL 注入防护**：使用参数化查询
- **XSS 防护**：输入验证和输出编码
- **CSRF 防护**：使用 CSRF Token

## 技术选型理由

1. **React + TypeScript**：提供类型安全和组件化开发，适合构建复杂的交易平台
2. **Node.js**：事件驱动，非阻塞 I/O，适合处理高并发请求
3. **PostgreSQL**：强大的事务支持和复杂查询能力，适合金融交易系统
4. **Redis**：高性能缓存，适合处理实时数据和分布式锁
5. **RabbitMQ**：可靠的消息队列，适合处理异步任务和系统解耦
6. **微服务架构**：提高系统可扩展性和可维护性，便于独立部署和升级

## 扩展性考虑

1. **水平扩展**：通过容器化和负载均衡，支持服务水平扩展
2. **数据分片**：针对大数据量的表，考虑水平分片
3. **服务降级**：在系统负载高时，自动降级非核心功能
4. **容灾备份**：定期备份数据，确保系统可靠性
5. **多区域部署**：支持多区域部署，提高系统可用性
