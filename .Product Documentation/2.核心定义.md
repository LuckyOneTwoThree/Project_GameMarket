## 一、核心定义

SKU定义表主要决定了我们的数据库**存什么。**

**目前先从CS2入手，**主要涵盖 **CS2 中最核心的三类资产**：

1. **硬通货** ：如 AK-47 红线等等，量大，做市商最爱。
2. **分级资产** ：低磨损精品，体现“交易所”的精细化定价能力。
3. **绝对标品** ：箱子/钥匙，纯金融属性。（待定）

## 三、相关规则表

需注意：各个规则标准仅作设计参考，并非最终确定的标准。

### 3.1 资金与状态定义

在执行具体规则前，必须明确以下定义：

1. **SLA** **时效标准（动态）**：
   1. 不执行固定时长，而是读取当前订单对应 SKU 的 “**sla_minutes** ” 字段。
   2. ***参考值：****Bot 现货通常为 1分钟，**P2P* *饰品通常为 30分钟，Web3 链上资产通常为 10分钟。*
2. **资金状态**：
   1.  **Frozen (冻结)**: 资金在买家账户，但被锁定，不可用。
   2. **Released (划转)**: 资金从冻结状态划入卖家/平台账户。
   3. **Refunded (退回)**: 解冻资金，回到买家余额。
   4. **Slashed (罚没)**: 从保证金/余额中扣除违约金，注入**赔付池**或支付给受害方。
3. **上游维护豁免**：
   1. **Steam**: 每周二 23:00 - 01:00 (UTC)。
   2. **Web3**: 链上 Gas 费飙升 (> 100 Gwei) 或 RPC 节点延迟 (> 5s)。
   3. ***动作：****触发上述任一条件，对应 GameID 的倒计时自动挂起，不触发违约判定。*

### 3.2 轨道 A：托管模式清算规则

核心原则：平台负全责。Bot 的错误由平台赔偿，用户的错误由用户买单。

代码异常场景技术判定依据 系统处理逻辑 资金流转 赔付标准 (SLA)C-01交付超时发起转账后，时间超过 sku.sla_minutes 且上游未确认成功。1. 调用 CancelTradeOffer。
\2. 若 Cancel 成功 → 订单关闭。全额退款无赔付(视为不可抗力)。C-02防双花回滚失败Bot 尝试 Cancel 报价，但上游返回 Error 或状态未知1.挂起订单 (SUSPEND)。
\2. 推送至人工审计队列。
\3. 严禁自动退款。资金持续冻结(直到人工确认货还在库里)视人工审计结果而定。C-03平台超卖/缺货撮合成功，但数据库行锁失败，或物理库存缺失。1. 立即撤销撮合。
\2. 下架该异常 SKU。全额退款 + 赔付金赔付买家 105%(5% 违约金由平台风险池支出)。C-04买家未接受/过期报价已发出，买家在 Expire_Time 前未确认。1. 资产回库。
\2. 记录买家信用污点。退款 - 罚金扣除买家 1% (Min 0.5U)(作为库存占用费)。C-05入库充值未到账用户发货给 Bot，Bot 已收货，但 DB 未增加余额。1. 用户提交工单 (附 TransactionID)。
\2. 客服核对 Steam History。人工补录余额若延迟 >24h，执行相关赔付

### 3.3 轨道 B：P2P 模式清算规则

核心原则：平台做裁判。基于“预言机（Oracle）”的链上/链下数据进行自动裁决。

代码异常场景技术判定依据 系统处理逻辑 资金流转 赔付标准 (SLA)P-00死单拦截撮合瞬间，Redis 中检测到商家 Heartbeat 丢失 (插件离线)。1. 不进行匹配。
\2. 将该商家挂单设为 Invisible无资金变动无。P-01卖家超时未发匹配后 30分钟 + 2分钟(宽限) 内，卖家未上传有效 Trade ID。1. 自动撤单。
\2. 扣除卖家信誉分。买家：全额退款 卖家：扣除保证金赔付买家 5 USDT(从卖家保证金中扣除)。P-02属性欺诈/错发预言机检测到发货物品的 HashName 或 Attributes 与订单不符。1. 预言机判定 MISMATCH。
\2. 冻结卖家挂单权限。买家：全额退款  卖家：罚没 20% 保证金赔付买家 10% 订单额(剩余罚金充入平台风险池)。P-03买家无法接收商家举证买家红锁/地址错误/库存满。1. 卖家提交报错截图凭证。
\2. 人工/OCR 审核通过。买家：退款 - 5% 违约金   卖家：收到违约金补偿卖家 5% 订单额(补偿卖家被锁仓/浪费时间的损失)。P-04买家恶意拒收预言机检测到上游状态为 主动拒绝。1. 判定交易失败。
\2. 记录买家恶意行为。买家：退款 - 2% 违约金   卖家：收到违约金补偿卖家 2% 订单额。P-05API 劫持防御卖家发货瞬间，Offer 被“秒取消” (Auto-Cancel 脚本)。1. 判定买家 API Key 泄露。
\2. 冻结买家账户保护资金。交易取消免责。强制买家重置 Steam API Key。

### 3.4 轨道 C：商户风控与强平规则

**核心原则**：保护平台资产负债表，防止商家穿仓（赔不起）。

\2. 强制下架所有挂单。邮件通知

### 3.5 轨道 D：系统性熔断规则

核心原则：保护平台资产负债表不被打穿。
