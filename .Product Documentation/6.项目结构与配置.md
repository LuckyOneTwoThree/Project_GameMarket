## 配置文件

### 前端配置

#### vite.config.js

```javascript
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'
import path from 'path'

export default defineConfig({
  plugins: [react()],
  resolve: {
    alias: {
      '@': path.resolve(__dirname, './src')
    }
  },
  server: {
    port: 3000,
    proxy: {
      '/api': {
        target: 'http://localhost:8000',
        changeOrigin: true,
        rewrite: (path) => path.replace(/^\/api/, '')
      }
    }
  }
})
```

#### .env 文件

```
# 前端环境变量
VITE_API_URL=http://localhost:8000
VITE_WS_URL=ws://localhost:8000
VITE_APP_TITLE=Game Market
VITE_APP_VERSION=1.0.0
```

### 后端配置

#### .env 文件

```
# 数据库配置
DATABASE_URL=postgresql://user:password@localhost:5432/gamemarket

# Redis 配置
REDIS_URL=redis://localhost:6379

# RabbitMQ 配置

# Steam API 配置
STEAM_API_KEY=your_steam_api_key
STEAM_BOT_USERNAME=your_bot_username

# JWT 配置

# 服务器配置
PORT=8000
NODE_ENV=development

# 安全配置
CORS_ORIGIN=http://localhost:3000
RATE_LIMIT_WINDOW_MS=15 * 60 * 1000
RATE_LIMIT_MAX=100
```

## 代码规范

### 前端代码规范

- 使用 ESLint + Prettier 进行代码检查和格式化
- 遵循 React 最佳实践，使用函数组件和 Hooks
- 使用 TypeScript 进行类型检查
- 组件命名使用 PascalCase，函数和变量使用 camelCase
- 文件名与组件名保持一致

### 后端代码规范

- 使用 ESLint 进行代码检查
- 遵循 Node.js 最佳实践
- 控制器负责处理 HTTP 请求和响应
- 服务层负责业务逻辑
- 模型层负责数据访问
- 中间件负责横切关注点
- 函数和变量使用 camelCase
- 文件名使用小写字母和连字符

## 开发工作流

1. **分支管理**：使用 Git Flow 工作流，主分支为 main，开发分支为 develop
2. **代码审查**：通过 Pull Request 进行代码审查
3. **持续集成**：使用 CI/CD 工具（如 GitHub Actions）进行自动构建和测试
4. **版本管理**：使用 Semantic Versioning 进行版本管理

## 部署流程

1. **前端部署**：构建生产版本，部署到 CDN 或静态网站托管服务
2. **后端部署**：构建生产版本，部署到云服务器或容器平台
3. **数据库部署**：使用 managed PostgreSQL 服务
4. **缓存部署**：使用 managed Redis 服务
5. **消息队列部署**：使用 managed RabbitMQ 服务

## 监控与维护

1. **系统监控**：使用 Prometheus + Grafana 监控系统性能
2. **日志管理**：使用 ELK Stack 集中管理日志
3. **告警机制**：设置关键指标告警，及时响应系统异常
4. **备份策略**：定期备份数据库和关键配置
